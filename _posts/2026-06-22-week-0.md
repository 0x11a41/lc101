### 1. [https://leetcode.com/problems/**two-sum**](https://leetcode.com/problems/two-sum/)
We are asked to find the indices of two numbers in an array that add up to a given
target value.

The easiest method would be to check if any pair of numbers in the array add up
to target.

```text
for (i = 0 to n)
	for (j = i + 1 to n)
		x = arr[i], y = arr[j]
		if (x + y == target)
			return (i, j)
```

This garunteed approach indeed will return the correct indices, but yields an
O(n<sup>2</sup>) time complexity, due the nested loops.

For further optimizing this, we need to analyze what is happening inside the loops
and try to come up with a clever way to minimize computation.

Inside those nested loops, the following comparison drives the entire logic.
```
x + y == target
```
which is an algebric formula. we can rearrange this formula as follows:
```
y = target - x
```
This means, by using value of *'x'*, we can calculate *'y'* from the above formula.
Now we only need to search where *'y'* is located inside the array. The inner loop
is precisely doing that; but it does so linearly. Therefore, we can optimize the
inner loop if we can come up with a better **searching strategy**.

Since we cannot rearrange the elements to perform a binary search, we can optimize
searching using a hash table structured as *(key = arr[i], value = i)*. A hashtable
is an array that stores elements as a key-value pair and uses a hash function to
mathematically generate the index of any key in constant time.

</div>
Instead of scanning the array repeatedly, we can search *"target - x"* inside the
hash table for obtaining position of *'y'*. We don't need the inner loop anymore.

- we start from one's place. This means we should start from last element in array.

- Whenever the sum of digits goes beyond 9, we add a carry to the next digit.

> *carry* can be calculated using integer division (*/ 10*), and the single
digit left behind at the sum[i] is calculated using the modulo operator (*% 10*).
```c
int x = nums1[i] + nums2[i] + carry;
carry  = x / 10;
sum[i] = x % 10;
```

The only edge case would be when carry variable still contain some value after
we finish adding all digits.

When doing addition by hand, we simply place that leftover
carry at the leftmost position of our result. We can translate that action into
program by inserting the remaining carry at the beginning of the array.

This approach will work for general addition. Since the question only asked us
add one to given number, we can optimize this process.

When adding one, a carry is born only when we encounder a nine. This means
we don't need complex math formulas. We can simply loop backward through the
array and apply two simple rules:

- If we see a 9, we turn it into a 0 and keep going.

### 3. [https://leetcode.com/problems/**move-zeros**](https://leetcode.com/problems/move-zeroes)
To solve this, we can create a new array, copy all non-zero values to it and zero-pad till the end. But the problem demands a strategy that works by rearranging the elements within the given array itself.

#### Approach #1
1. Iterate over all elements starting from first element of the array.
2. Whenever a zero is found, repeatdly swap it to the end.
